## 日志处理流程

**文件**: `Pages/LaunchPage.xaml.cs` (第261-304行)

### 1. 订阅事件

```csharp
_gameLauncher.LaunchLogReceived += OnLaunchLogReceived;
```

### 2. 接收日志

```csharp
private void OnLaunchLogReceived(string log)
{
    lock (_logBuffer)
    {
        _logBuffer.Enqueue(log);  // 加入缓冲队列
    }
    ProcessLogBuffer();
}
```

### 3. 显示日志

```csharp
private void ProcessLogBuffer()
{
    Dispatcher.Invoke(() =>
    {
        while (_logBuffer.Count > 0)
        {
            string log = _logBuffer.Dequeue();
            txtLog.Text += $"[{DateTime.Now:HH:mm:ss}] {log}";
            
            // 限制日志长度（超过100000字符自动裁剪）
            if (txtLog.Text.Length > 100000)
            {
                // 裁剪前50000字符
                int overflowIndex = txtLog.Text.IndexOf(Environment.NewLine, 50000);
                if (overflowIndex > 0)
                {
                    txtLog.Text = txtLog.Text.Substring(overflowIndex + Environment.NewLine.Length);
                }
                else
                {
                    txtLog.Text = txtLog.Text.Substring(50000);
                }
            }
        }
    });
}
```

## 特性

- ✅ **实时显示**：通过事件回调实时接收并显示日志
- ✅ **带时间戳**：每条日志自动添加HH:mm:ss格式的时间戳
- ✅ **可滚动查看**：支持垂直和水平滚动，方便查看长日志
- ✅ **自动裁剪**：超过100000字符自动清理旧日志，防止内存溢出
- ✅ **只读模式**：用户无法编辑日志内容
- ✅ **等宽字体**：使用Consolas字体，适合查看日志和技术信息
- ✅ **缓冲队列**：使用队列缓冲日志，避免UI线程阻塞
- ✅ **线程安全**：使用lock确保多线程环境下的安全性

## 日志标签说明

| 标签 | 说明 |
|------|------|
| `[Java]` | Java路径检测、版本信息 |
| `[Natives]` | 原生库提取、解压 |
| `[CP]` | 类路径构建 |
| `[修复]` | 依赖库修复、下载 |
| `[校验]` | 文件完整性校验 |
| `[参数]` | 启动参数处理 |
| `[参数校验]` | 参数验证 |
| `[启动命令]` | 完整启动命令 |
| `[ERROR]` | 错误信息 |
| `[警告]` | 警告信息 |
